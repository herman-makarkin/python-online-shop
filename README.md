### Практическая работа по ООП в Python: Система управления онлайн-магазином

#### Цель работы
Закрепить понимание основных принципов объектно-ориентированного программирования: создание классов и объектов, инкапсуляция, наследование, полиморфизм. Научиться проектировать взаимосвязанные классы для моделирования реального процесса — работы интернет-магазина.

#### Задание
Разработайте программу для управления небольшим онлайн-магазином. Программа должна позволять просматривать товары, добавлять их в корзину, оформлять заказы, управлять скидками и отслеживать статус заказа.

##### Базовые требования 
Реализуйте следующие классы:

1. **Класс `Product`** (товар):
   - Атрибуты: `name` (название), `price` (цена), `quantity` (количество на складе), `product_id` (уникальный идентификатор).
   - Методы:
     - `__init__(self, name, price, quantity, product_id)` – конструктор.
     - `__str__(self)` – возвращает строку с информацией о товаре.
     - Геттеры и сеттеры для `price` и `quantity` (через `@property`), с проверкой, что цена положительная, а количество неотрицательное.
     - `is_available(self, needed_quantity)` – проверяет, достаточно ли товара на складе для заказа нужного количества.

2. **Класс `Customer`** (покупатель):
   - Атрибуты: `name` (имя), `customer_id` (уникальный), `email` (электронная почта).
   - Методы:
     - `__init__(self, name, customer_id, email)`.
     - `__str__(self)` – возвращает строку с именем и email.
     - Геттеры/сеттеры для email с проверкой формата (можно упрощённо – проверять наличие символа '@').

3. **Класс `Cart`** (корзина покупателя):
   - Атрибуты: `customer` (объект `Customer`), `items` (словарь: `{product_id: quantity}`).
   - Методы:
     - `__init__(self, customer)` – привязывает корзину к покупателю.
     - `add_item(self, product, quantity)` – добавляет товар в корзину (если есть на складе). Если товар уже есть, увеличивает количество.
     - `remove_item(self, product_id)` – удаляет товар из корзины.
     - `update_quantity(self, product_id, new_quantity)` – изменяет количество товара в корзине.
     - `get_total_price(self, product_catalog)` – вычисляет общую стоимость товаров в корзине (принимает словарь товаров магазина для получения цен).
     - `clear(self)` – очищает корзину.
     - `__str__(self)` – возвращает содержимое корзины.

4. **Класс `Order`** (заказ):
   - Атрибуты: `order_id` (уникальный), `customer` (объект `Customer`), `items` (копия состава корзины), `total_price` (общая стоимость), `status` (статус заказа: "новый", "оплачен", "отправлен", "доставлен").
   - Методы:
     - `__init__(self, order_id, customer, items, total_price)` – конструктор, статус по умолчанию "новый".
     - `update_status(self, new_status)` – изменяет статус (с проверкой допустимости перехода).
     - `__str__(self)` – возвращает информацию о заказе.

5. **Класс `Shop`** (магазин):
   - Атрибуты: `products` (словарь товаров: `{product_id: Product}`), `customers` (словарь покупателей), `orders` (список заказов), `carts` (словарь корзин: `{customer_id: Cart}`).
   - Методы:
     - `add_product(self, product)` – добавляет товар в ассортимент.
     - `register_customer(self, customer)` – регистрирует покупателя.
     - `get_cart(self, customer)` – возвращает корзину для данного покупателя (если её нет, создаёт новую).
     - `create_order(self, customer)` – создаёт заказ из корзины покупателя, уменьшает количество товаров на складе, очищает корзину и добавляет заказ в список.
     - `find_products_by_name(self, name)` – ищет товары по части названия (без учёта регистра).
     - `display_products(self)` – выводит список всех товаров с ценами и остатками.

Реализуйте наследование и полиморфизм:

- Создайте подкласс `DigitalProduct`, наследующий от `Product`. Добавьте атрибут `file_size` (размер файла в МБ) и `download_link`. Переопределите `__str__` и метод `is_available` (цифровой товар всегда доступен, если количество не задано или достаточно лицензий).
- Создайте подкласс `DiscountedProduct`, наследующий от `Product`. Добавьте атрибут `discount_percent` и переопределите геттер `price`, чтобы возвращать цену со скидкой. При изменении цены скидка пересчитывается.
- Реализуйте интерфейс для разных способов оплаты (например, класс `PaymentProcessor` с методом `process_payment(order, amount)`) и его подклассы `CreditCardProcessor`, `PayPalProcessor` (имитация обработки). Модифицируйте `Shop.create_order`, чтобы можно было передать способ оплаты и вызвать соответствующий процессор.

- Реализуйте магические методы для класса `Shop`: `__len__` (количество товаров), `__getitem__` (доступ к товару по индексу или product_id).
- Добавьте возможность сохранения и загрузки данных (товаров, покупателей, заказов) в JSON-файл.
- Реализуйте логирование действий (добавление в корзину, оформление заказа) в отдельный файл.


#### Требования к сдаче
- Код должен быть оформлен в одном или нескольких файлах `.py`.
- Код должен соответствовать PEP 8.
- Обязательно продемонстрировать работу программы на примере (можно скопировать пример выше и дополнить своими тестами).

#### Критерии оценки
- **3 (удовлетворительно)**: реализованы базовые классы `Product`, `Customer`, `Cart`, `Order`, `Shop` с основными методами. Корректно работают добавление в корзину, оформление заказа, проверка наличия. Код не содержит грубых ошибок.
- **4 (хорошо)**: выполнены базовые требования + дополнительное задание (хотя бы один подкласс или дополнительный функционал). Используются свойства, проверки данных.
- **5 (отлично)**: реализованы все дополнительные задания, код хорошо структурирован, применены принципы ООП (инкапсуляция, полиморфизм), добавлена обработка исключений, пример демонстрирует все возможности.

#### Советы
- Продумайте, какие атрибуты должны быть приватными. Используйте свойства для доступа к ним.
- При добавлении товара в корзину проверяйте наличие на складе; при создании заказа списывайте количество.
- Не забывайте про уникальность идентификаторов (можно использовать простые строки или генерацию).
- Для поиска по названию можно использовать `in` и приведение к нижнему регистру.
- При реализации скидок учтите, что цена со скидкой не должна быть отрицательной.
